#41 缺失的第一个正数

给你一个未排序的整数数组 nums，请你找出其中没有出现的最小的正整数。要求算法的时间复杂度为 O(n)，并且只能使用常数级别的额外空间。

示例 1
输入：nums = [1,2,0]
输出：3
示例 2
输入：nums = [3,4,-1,1]
输出：2
示例 3
输入：nums = [7,8,9,11,12]
输出：1
2 题目拆解
未排序数组 ➜ 找最小缺失正数 ➜ O(n) + O(1)空间 ➜ 返回结果

关键约束：

  • 时间复杂度 O(n) → 不能排序
  • 空间复杂度 O(1) → 不能用额外哈希表
  • 答案范围一定是 [1, n+1]，其中 n 为数组长度
  • 利用数组本身做哈希表 → 原地哈希
3 解题思路

核心洞察：长度为 n 的数组，缺失的最小正数一定在 [1, n+1] 之间。

原地哈希法：把每个值 v 放到下标 v-1 的位置，即让 nums[i] = i+1。遍历完成后，第一个 nums[i] ≠ i+1 的位置，答案就是 i+1。

遍历数组 ➜ 将 v 交换到 v-1 位置 ➜ 跳过越界/重复 ➜ 找第一个 nums[i]≠i+1

交换条件：当 1 ≤ nums[i] ≤ n 且 nums[i] ≠ nums[nums[i]-1] 时，交换 nums[i] 和 nums[nums[i]-1]。

4 代码演示
def firstMissingPositive(nums): n = len(nums) # 原地哈希：把值 v 放到下标 v-1 for i in range(n): while 1 <= nums[i] <= n and nums[i] != nums[nums[i]-1]: # 交换 nums[i] 和 nums[nums[i]-1] j = nums[i] - 1 nums[i], nums[j] = nums[j], nums[i] # 找第一个不在正确位置的 for i in range(n): if nums[i] != i + 1: return i + 1 return n + 1
public int firstMissingPositive(int[] nums) { int n = nums.length; for (int i = 0; i < n; i++) { while (nums[i] >= 1 && nums[i] <= n && nums[i] != nums[nums[i] - 1]) { int j = nums[i] - 1; int tmp = nums[i]; nums[i] = nums[j]; nums[j] = tmp; } } for (int i = 0; i < n; i++) { if (nums[i] != i + 1) return i + 1; } return n + 1; }
int firstMissingPositive(vector<int>& nums) { int n = nums.size(); for (int i = 0; i < n; i++) { while (nums[i] >= 1 && nums[i] <= n && nums[i] != nums[nums[i] - 1]) { swap(nums[i], nums[nums[i] - 1]); } } for (int i = 0; i < n; i++) { if (nums[i] != i + 1) return i + 1; } return n + 1; }

6 易错要点
用 while 不是 if

交换后 nums[i] 变了新值，可能仍需继续交换，必须用 while 循环直到当前位置放好或越界。

防重复死循环

条件 nums[i] != nums[nums[i]-1] 防止相同值反复交换。如 [1,1]，不加此条件会死循环。

答案范围

答案一定是 [1, n+1]。若 1~n 都出现了，答案为 n+1；否则第一个缺失的就是答案。

7 扩展思考
  • 如果允许 O(n) 空间：直接用哈希集合存所有正数，再从 1 开始查，代码更简单
  • 如果允许排序：排序后扫描，O(n log n) 时间但思路更直观
  • 标记法替代：第一遍将负数和零改为 n+1，第二遍将 |nums[i]|-1 位置取负，第三遍找第一个正数位置
  • 推广：缺失的第 k 个正数 → LeetCode #1539，二分或模拟
