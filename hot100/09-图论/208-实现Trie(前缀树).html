#208 实现Trie(前缀树)

Trie（发音类似 "try"）或者说前缀树，是一种树形数据结构，用于高效地存储和检索字符串数据集中的键。实现 Trie 类：insert(word) 向 Trie 中插入字符串，search(word) 查找字符串，startsWith(prefix) 查找前缀。

示例
输入：["Trie","insert","search","startsWith"] [[],["apple"],["apple"],["app"]]
输出：[null,null,true,true]
插入 "apple" 后，search("apple") 返回 true，startsWith("app") 返回 true
补充
输入：search("app")
输出：false
"app" 只是前缀，不是完整单词，search 返回 false
2 题目拆解
根节点(空) ➜ 每条边=一个字符 ➜ 节点标记isEnd ➜ 插入/查找/前缀

关键约束：

  • 每个节点最多有 26 个子节点（小写英文字母）
  • search 要求完整单词匹配（isEnd = true）
  • startsWith 只需前缀存在即可，不要求 isEnd
3 解题思路

核心数据结构：每个节点包含 children[26] 数组和 isEnd 布尔标记。

insert：逐字符走，不存在则创建 ➜ search：逐字符走，走通且isEnd ➜ startsWith：逐字符走，走通即可

时间复杂度：每个操作 O(L)，L 为字符串长度。空间：O(N×L)，N 为单词数。

class Trie: def __init__(self): self.children = {} # 字符→子节点 self.is_end = False def insert(self, word): node = self for ch in word: if ch not in node.children: node.children[ch] = Trie() node = node.children[ch] node.is_end = True def search(self, word): node = self._search_prefix(word) return node is not None and node.is_end def startsWith(self, prefix): return self._search_prefix(prefix) is not None def _search_prefix(self, prefix): node = self for ch in prefix: if ch not in node.children: return None node = node.children[ch] return node

6 易错要点
isEnd 标记

search 必须检查 isEnd！插入 "apple" 后 search("app") 返回 false，因为 "app" 节点的 isEnd 为 false。

startsWith vs search

startsWith 只需前缀路径存在，不检查 isEnd。search 需要路径存在且 isEnd = true。

创建 vs 复用

insert 时，子节点存在就复用，不存在才创建。这是 Trie 节省空间的关键——公共前缀共享节点。

7 扩展思考
  • 自动补全：在 Trie 节点存储子树单词列表，或 DFS 遍历子树
  • 单词搜索 II：回溯 + Trie 剪枝，避免重复搜索
  • 压缩Trie (Radix Tree)：合并单分支路径，减少节点数
  • 后缀树/后缀自动机：更高级的字符串数据结构，处理更复杂模式匹配
