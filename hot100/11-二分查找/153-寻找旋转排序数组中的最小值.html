#153 寻找旋转排序数组中的最小值

已知一个长度为 n 的数组，预先按照升序排列，经由 1 到 n 次旋转后，得到输入数组。请找出数组中的最小元素。要求时间复杂度 O(log n)。

示例 1
输入：nums = [3,4,5,1,2]
输出：1
示例 2
输入：nums = [4,5,6,7,0,1,2]
输出：0
示例 3
输入：nums = [11,13,15,17]
输出：11
2 题目拆解
旋转排序数组 ➜ 最小值在旋转点 ➜ 二分找旋转点 ➜ 返回最小值

关键约束：

  • 数组由两段升序拼接，最小值就在旋转点
  • 元素互不相同，不存在等于边界的情况
  • 可能没有旋转（即旋转 n 次），此时最小值就是 nums[0]
3 解题思路

核心思想：用 nums[mid] 与 nums[right] 比较。若 nums[mid] > nums[right]，说明最小值在 mid 右边；否则最小值在 mid 及其左边。

取 mid ➜ nums[mid] > nums[right]? ➜ 是 → left=mid+1 ➜ 否 → right=mid

为什么和 nums[right] 比？右边界是「当前搜索区间」的最大值，如果 mid 比它大，说明 mid 还在左半段（大段），最小值一定在右边。

4 代码演示
def findMin(nums): left, right = 0, len(nums) - 1 while left < right: # 注意：left < right，不是 <= mid = (left + right) // 2 if nums[mid] > nums[right]: left = mid + 1 # 最小值在右半 else: right = mid # 最小值在 mid 或左半 return nums[left]
public int findMin(int[] nums) { int left = 0, right = nums.length - 1; while (left < right) { int mid = left + (right - left) / 2; if (nums[mid] > nums[right]) { left = mid + 1; } else { right = mid; } } return nums[left]; }
int findMin(vector<int>& nums) { int left = 0, right = nums.size() - 1; while (left < right) { int mid = left + (right - left) / 2; if (nums[mid] > nums[right]) { left = mid + 1; } else { right = mid; } } return nums[left]; }

6 易错要点
循环条件 left < right

本题用 left < right，不是 left <= right。当 left == right 时已经找到最小值，不需要再循环。且 right = mid 配合 < 才能终止。

right = mid 不是 mid-1

当 nums[mid] <= nums[right] 时，mid 本身可能就是最小值，所以 right = mid，不能跳过 mid。

和 nums[right] 比而非 nums[left]

和 nums[left] 比无法区分「未旋转」和「mid 在左半」两种情况。和 nums[right] 比逻辑更清晰。
