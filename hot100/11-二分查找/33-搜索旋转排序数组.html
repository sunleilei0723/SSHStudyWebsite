#33 搜索旋转排序数组

整数数组 nums 按升序排列（值互不相同），在某个下标 k 处进行了旋转。给你旋转后的数组 nums 和目标值 target，如果 target 存在则返回其下标，否则返回 -1。算法时间复杂度 O(log n)。

示例 1
输入：nums = [4,5,6,7,0,1,2], target = 0
输出：4
示例 2
输入：nums = [4,5,6,7,0,1,2], target = 3
输出：-1
2 题目拆解
旋转排序数组 ➜ 两段各自有序 ➜ 判断 mid 在哪段 ➜ 在有序段中二分

关键约束：

  • 数组由两段升序子数组拼接而成，旋转点处有断崖
  • 元素互不相同，不存在重复干扰判断
  • 必须 O(log n)，不能线性扫描
3 解题思路

核心思想：二分时，mid 必将数组分成两半，其中一半一定是有序的。先判断哪半有序，再判断 target 是否在有序半边，决定搜索方向。

取 mid ➜ nums[left] ≤ nums[mid]? 左半有序 ➜ target 在有序半? ➜ 收缩到对应半边

判断逻辑：

  • 若 nums[left] <= nums[mid]：左半有序 → 看 target 是否在 [nums[left], nums[mid]) 中
  • 否则：右半有序 → 看 target 是否在 (nums[mid], nums[right]] 中
4 代码演示
def search(nums, target): left, right = 0, len(nums) - 1 while left <= right: mid = (left + right) // 2 if nums[mid] == target: return mid if nums[left] <= nums[mid]: # 左半有序 if nums[left] <= target < nums[mid]: right = mid - 1 else: left = mid + 1 else: # 右半有序 if nums[mid] < target <= nums[right]: left = mid + 1 else: right = mid - 1 return -1
public int search(int[] nums, int target) { int left = 0, right = nums.length - 1; while (left <= right) { int mid = left + (right - left) / 2; if (nums[mid] == target) return mid; if (nums[left] <= nums[mid]) { if (nums[left] <= target && target < nums[mid]) right = mid - 1; else left = mid + 1; } else { if (nums[mid] < target && target <= nums[right]) left = mid + 1; else right = mid - 1; } } return -1; }
int search(vector<int>& nums, int target) { int left = 0, right = nums.size() - 1; while (left <= right) { int mid = left + (right - left) / 2; if (nums[mid] == target) return mid; if (nums[left] <= nums[mid]) { if (nums[left] <= target && target < nums[mid]) right = mid - 1; else left = mid + 1; } else { if (nums[mid] < target && target <= nums[right]) left = mid + 1; else right = mid - 1; } } return -1; }

6 易错要点
nums[left] ≤ nums[mid] 用 ≤

当 left == mid 时（只有两个元素），左半也算「有序」，必须用 ≤。用 < 会漏判。

先判有序半，再判 target

必须先确定哪半有序，再判断 target 是否落在有序半中。顺序不能反，否则逻辑混乱。

有重复元素时失效

当 nums[left] == nums[mid] 且 left != mid 时无法判断哪半有序，需要特殊处理（如 left++），见 #81。

7 扩展思考
  • 81. 搜索旋转排序数组 II：允许重复元素，nums[left]==nums[mid] 时无法判断，需 left++ 退化
  • 先找旋转点再二分：也可先用二分找最小值位置（旋转点），再在对应半边做标准二分
  • 多次查询优化：先找旋转点 O(log n)，之后每次查询 O(log n)，适合批量查询
