#04 寻找两个正序数组的中位数

给定两个大小分别为 m 和 n 的正序（从小到大）数组 nums1 和 nums2，找出并返回这两个正序数组的中位数。算法时间复杂度应为 O(log(m+n))。

示例 1
输入：nums1 = [1,3], nums2 = [2]
输出：2.00000
合并数组 = [1,2,3]，中位数 2
示例 2
输入：nums1 = [1,2], nums2 = [3,4]
输出：2.50000
合并数组 = [1,2,3,4]，中位数 (2+3)/2 = 2.5
2 题目拆解
两个有序数组 ➜ 找第 k 小元素 ➜ 二分排除法 ➜ 计算中位数

关键约束：

  • 两个数组分别有序，但不合并
  • 要求 O(log(m+n))，不能合并后排序
  • 中位数 = 第 (m+n+1)/2 小 和 第 (m+n+2)/2 小的平均值
3 解题思路

核心思想：转化为「找第 k 小元素」问题。每次比较两个数组中第 k/2 个元素，较小的那个及其前面的元素一定不是第 k 小，可以排除。

找第 k 小 ➜ 比较 A[k/2] 和 B[k/2] ➜ 排除较小的一半 ➜ k 减半，递归

边界情况：某个数组为空时直接取另一个数组的第 k 个；k=1 时取两个数组首元素较小者。

4 代码演示
def findMedianSortedArrays(nums1, nums2): def kth(a, b, k): if not a: return b[k-1] if not b: return a[k-1] if k == 1: return min(a[0], b[0]) i = min(len(a), k//2) - 1 j = min(len(b), k//2) - 1 if a[i] <= b[j]: return kth(a[i+1:], b, k-(i+1)) else: return kth(a, b[j+1:], k-(j+1)) m, n = len(nums1), len(nums2) left = kth(nums1, nums2, (m+n+1)//2) right = kth(nums1, nums2, (m+n+2)//2) return (left + right) / 2
public double findMedianSortedArrays(int[] nums1, int[] nums2) { int m = nums1.length, n = nums2.length; int left = kth(nums1, 0, nums2, 0, (m+n+1)/2); int right = kth(nums1, 0, nums2, 0, (m+n+2)/2); return (left + right) / 2.0; } int kth(int[] a, int sa, int[] b, int sb, int k) { if (sa >= a.length) return b[sb+k-1]; if (sb >= b.length) return a[sa+k-1]; if (k == 1) return Math.min(a[sa], b[sb]); int i = Math.min(a.length, sa+k/2) - 1; int j = Math.min(b.length, sb+k/2) - 1; if (a[i] <= b[j]) return kth(a, i+1, b, sb, k-(i-sa+1)); else return kth(a, sa, b, j+1, k-(j-sb+1)); }
double findMedianSortedArrays(vector<int>& a, vector<int>& b) { int m = a.size(), n = b.size(); auto kth = [&](auto& f, int sa, int sb, int k) -> int { if (sa == m) return b[sb+k-1]; if (sb == n) return a[sa+k-1]; if (k == 1) return min(a[sa], b[sb]); int i = min(m, sa+k/2) - 1, j = min(n, sb+k/2) - 1; if (a[i] <= b[j]) return f(f, i+1, sb, k-(i-sa+1)); else return f(f, sa, j+1, k-(j-sb+1)); }; int l = kth(kth,0,0,(m+n+1)/2); int r = kth(kth,0,0,(m+n+2)/2); return (l + r) / 2.0; }

6 易错要点
数组越界保护

取 k/2 时可能超出数组长度，必须 min(len, k/2)。否则访问越界直接报错。

排除数量计算

排除 a[0..i] 共 i+1 个元素，所以 k 减去 i-sa+1，不是 k/2。因为数组可能不够长。

奇偶统一处理

用 (m+n+1)/2 和 (m+n+2)/2 两次求第 k 小再取平均，无论奇偶都适用，避免分情况讨论。

7 扩展思考
  • 划分数组法：另一种 O(log(min(m,n))) 解法，在较短数组上二分划分位置，保证左边元素数 = 右边元素数
  • 找任意第 k 小：本算法可推广为「两个有序数组找第 k 小」，不限于中位数
  • 多个有序数组：3 个以上有序数组找第 k 小，可用堆或分治策略，但复杂度更高
