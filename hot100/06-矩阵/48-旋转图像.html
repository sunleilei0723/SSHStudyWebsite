#48 旋转图像

给定一个 n × n 的二维矩阵 matrix 表示一个图像，请你将图像顺时针旋转 90 度。你必须在原地旋转图像，即直接修改输入的矩阵，不要使用另一个矩阵来旋转。

示例 1
输入：matrix = [[1,2,3],[4,5,6],[7,8,9]]
输出：[[7,4,1],[8,5,2],[9,6,3]]
示例 2
输入：matrix = [[5,1,9,11],[2,4,8,10],[13,3,6,7],[15,14,12,16]]
输出：[[15,13,2,5],[14,3,4,1],[12,6,8,9],[16,7,10,11]]
2 题目拆解
n×n矩阵 ➜ 顺时针90° ➜ 原地修改 ➜ 返回void

关键约束：

  • 必须是原地操作，O(1) 额外空间
  • 矩阵是方阵（n × n），行列数相等
  • 旋转 90° 等价于 先转置 + 再水平翻转
3 解题思路

方法一：转置 + 翻转 O(n²)

顺时针旋转 90° = 沿主对角线转置 + 每行水平翻转。数学证明：matrix[i][j] → 转置后 matrix[j][i] → 翻转后 matrix[j][n-1-i]，恰好等于旋转 90° 后的位置。

沿对角线转置 ➜ 每行水平翻转 ➜ 完成旋转90°

方法二：四元素一轮换 O(n²)

每次旋转 4 个元素：[i][j] → [j][n-1-i] → [n-1-i][n-1-j] → [n-1-j][i] → [i][j]，一圈完成。只需处理左上 1/4 区域。

4 代码演示
def rotate(matrix): n = len(matrix) # 第一步：沿主对角线转置 for i in range(n): for j in range(i + 1, n): matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j] # 第二步：每行水平翻转 for row in matrix: row.reverse()
public void rotate(int[][] matrix) { int n = matrix.length; // 第一步：沿主对角线转置 for (int i = 0; i < n; i++) { for (int j = i + 1; j < n; j++) { int tmp = matrix[i][j]; matrix[i][j] = matrix[j][i]; matrix[j][i] = tmp; } } // 第二步：每行水平翻转 for (int i = 0; i < n; i++) { int l = 0, r = n - 1; while (l < r) { int tmp = matrix[i][l]; matrix[i][l] = matrix[i][r]; matrix[i][r] = tmp; l++; r--; } } }
void rotate(vector<vector<int>>& matrix) { int n = matrix.size(); // 第一步：沿主对角线转置 for (int i = 0; i < n; i++) for (int j = i + 1; j < n; j++) swap(matrix[i][j], matrix[j][i]); // 第二步：每行水平翻转 for (auto& row : matrix) reverse(row.begin(), row.end()); }

6 易错要点
转置只遍历上三角

转置时内层循环从 j = i+1 开始，不能从 0 开始，否则会交换两次又换回来。

翻转是水平不是垂直

第二步是每行水平翻转（左右互换），不是上下翻转。搞反方向会导致旋转方向错误。

逆时针旋转

逆时针 90° = 转置 + 垂直翻转（每列上下互换），注意区别。

7 扩展思考
  • 旋转 180°：等价于水平翻转 + 垂直翻转，不需要转置
  • 旋转 270°：等价于逆时针 90°，即转置 + 垂直翻转
  • 非方阵旋转：M×N 矩阵旋转后变成 N×M，必须用辅助数组，无法原地操作
  • 四元素一轮换法：一次处理 4 个对称位置，空间 O(1)，但代码稍复杂
