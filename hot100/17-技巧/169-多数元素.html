#169 多数元素

给定一个大小为 n 的数组 nums，找到其中的多数元素。多数元素是指在数组中出现次数大于 ⌊n/2⌋ 的元素。你可以假设数组是非空的，且给定的数组总是存在多数元素。

示例 1
输入：nums = [3,2,3]
输出：3
示例 2
输入：nums = [2,2,1,1,1,2,2]
输出：2
2 题目拆解
数组 nums ➜ 出现 > n/2 次 ➜ 找多数元素 ➜ O(n) O(1)

关键约束：

  • 多数元素出现次数严格大于 ⌊n/2⌋
  • 保证存在多数元素
  • 要求 O(n) 时间和 O(1) 空间
3 解题思路

摩尔投票法：想象一个"投票"场景，多数元素票数过半：

候选者 = nums[0] ➜ 相同 → count++ ➜ 不同 → count-- ➜ count=0 换人

核心思想：多数元素的数量比其他所有元素加起来还多，所以"抵消"后剩下的一定是多数元素。

4 代码演示
def majorityElement(nums): candidate = count = 0 for num in nums: if count == 0: candidate = num # 换候选人 count += (1 if num == candidate else -1) return candidate
public int majorityElement(int[] nums) { int candidate = 0, count = 0; for (int num : nums) { if (count == 0) candidate = num; count += (num == candidate) ? 1 : -1; } return candidate; }
int majorityElement(vector<int>& nums) { int candidate = 0, count = 0; for (int num : nums) { if (count == 0) candidate = num; count += (num == candidate) ? 1 : -1; } return candidate; }

6 易错要点
count=0 时换人

当 count 减到 0 时，当前候选人被"淘汰"，下一个元素成为新候选人。顺序不能反。

先判断 count==0

必须先检查 count==0 再决定是否换人，然后再更新 count。如果先更新 count 再换人就错了。

保证存在多数元素

题目保证多数元素存在，所以不需要二次验证。如果不保证，需要再遍历一次统计频次。

7 扩展思考
  • 出现 > n/3 的元素：用两个候选人，扩展摩尔投票法
  • 不保证存在：投票结束后再遍历一次验证频次是否真的过半
  • 排序法：排序后 nums[n/2] 一定是多数元素，但 O(n log n)
  • 哈希法：统计频次，O(n) 时间但 O(n) 空间
