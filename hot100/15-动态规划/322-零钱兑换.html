#322 零钱兑换

给你一个整数数组 coins 表示不同面额的硬币，另给一个整数 amount 表示总金额。计算并返回可以凑成总金额所需的最少的硬币个数。如果没有任何一种硬币组合能组成总金额，返回 -1。每种硬币的数量是无限的。

示例 1
输入：coins = [1,2,5], amount = 11
输出：3
11 = 5 + 5 + 1，共3枚硬币
示例 2
输入：coins = [2], amount = 3
输出：-1
无法用面额2的硬币凑出3
2 题目拆解
硬币面额 coins ➜ 目标金额 amount ➜ 最少硬币数 ➜ 不可达返回 -1

关键约束：

  • 每种硬币无限使用 → 完全背包
  • 求最少数量，不是方案数
  • 可能无法凑出，需返回 -1
3 解题思路

DP O(amount × |coins|)：定义 dp[i] = 凑出金额 i 需要的最少硬币数：

dp[0] = 0 ➜ dp[i] = min(dp[i-coin] + 1) ➜ 遍历所有 coin ≤ i

如果 dp[amount] 仍为 inf，说明无法凑出，返回 -1。

4 代码演示
def coinChange(coins, amount): dp = [float('inf')] * (amount + 1) dp[0] = 0 for i in range(1, amount + 1): for c in coins: if c <= i: dp[i] = min(dp[i], dp[i - c] + 1) return dp[amount] if dp[amount] != float('inf') else -1
public int coinChange(int[] coins, int amount) { int[] dp = new int[amount + 1]; Arrays.fill(dp, amount + 1); dp[0] = 0; for (int i = 1; i <= amount; i++) { for (int c : coins) { if (c <= i) dp[i] = Math.min(dp[i], dp[i - c] + 1); } } return dp[amount] > amount ? -1 : dp[amount]; }
int coinChange(vector<int>& coins, int amount) { vector<int> dp(amount + 1, amount + 1); dp[0] = 0; for (int i = 1; i <= amount; i++) { for (int c : coins) { if (c <= i) dp[i] = min(dp[i], dp[i - c] + 1); } } return dp[amount] > amount ? -1 : dp[amount]; }

6 易错要点
初始化技巧

Java/C++ 中可用 amount+1 代替 inf，因为最多用 amount 个1元硬币，超过此值即不可达。

返回 -1

如果 dp[amount] 仍为初始值（inf 或 amount+1），说明无法凑出，返回 -1。

硬币顺序无关

完全背包中硬币遍历顺序不影响结果，因为每枚硬币可无限使用。

7 扩展思考
  • 求组合方案数：LeetCode 518 零钱兑换 II，将 min 改为 sum
  • 每种硬币只能用一次：变为 0-1 背包，内层循环逆序
  • BFS 解法：将问题转化为图最短路径，从 amount BFS 到 0
  • 贪心+DFS：先按面额从大到小贪心选，再用 DFS 回溯，可剪枝优化
