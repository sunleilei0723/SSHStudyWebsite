#53 最大子数组和

给你一个整数数组 nums，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

2 题目拆解
输入数组 ➜ 逐个累加 ➜ 负和则重置 ➜ 记录最大值

关键约束：

  • 子数组必须是连续的
  • 至少包含一个元素，不能为空
  • 数组可能全为负数，此时返回最大负数
3 解题思路

暴力法 O(n²)：枚举所有子数组 → 太慢，淘汰

动态规划 / 贪心 O(n)：维护一个变量 cur 表示以当前元素结尾的最大子数组和。核心思想：如果 cur 变成负数，对后续只有拖累，不如从下一个元素重新开始：

遍历每个元素 ➜ cur = max(num, cur+num) ➜ ans = max(ans, cur) ➜ 返回 ans
4 代码演示
def maxSubArray(nums): cur = ans = nums[0] for num in nums[1:]: cur = max(num, cur + num) # 续上 or 重新开始 ans = max(ans, cur) # 更新全局最大 return ans
public int maxSubArray(int[] nums) { int cur = nums[0], ans = nums[0]; for (int i = 1; i < nums.length; i++) { cur = Math.max(nums[i], cur + nums[i]); ans = Math.max(ans, cur); } return ans; }
int maxSubArray(vector<int>& nums) { int cur = nums[0], ans = nums[0]; for (int i = 1; i < nums.size(); i++) { cur = max(nums[i], cur + nums[i]); ans = max(ans, cur); } return ans; }

6 易错要点
初始化不能为0

cur 和 ans 必须初始化为 nums[0]，不能为0。因为数组可能全为负数，最大和也是负数。

cur负数则重置

当 cur + num 不如 num 本身大时，说明之前的累加是负收益，应从 num 重新开始。

至少一个元素

子数组不能为空。如果题目允许空子数组（和为0），答案可能不同，要注意审题。

7 扩展思考
  • 分治法 O(n log n)：最大子数组要么在左半，要么在右半，要么跨越中点，可递归求解
  • 要求返回子数组位置：额外维护起止下标，在 cur 重置时更新起点
  • 环形数组版本：LeetCode 918，最大和可能是中间段或两端段
  • 乘积最大子数组：LeetCode 152，需同时维护最大和最小（负负得正）
