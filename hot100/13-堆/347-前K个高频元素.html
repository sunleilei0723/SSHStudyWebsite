#347 前K个高频元素

给你一个整数数组 nums 和一个整数 k ，请你返回其中出现频率前 k 高的元素。你可以按任意顺序返回答案。你的算法时间复杂度必须优于 O(n log n)。

示例
输入：nums = [1,1,1,2,2,3], k = 2
输出：[1,2]
1出现3次，2出现2次，3出现1次。前2高频是1和2
2 题目拆解
统计频率 ➜ 取前K高频 ➜ 小顶堆维护 ➜ 返回堆中元素

关键约束：

  • 时间复杂度优于 O(nlogn)，不能直接排序
  • 返回的是元素值，不是频率
  • 答案顺序任意
3 解题思路

两步走：先用哈希表统计频率，再用小顶堆取 TopK。

遍历统计频率 ➜ 建小顶堆(大小k) ➜ 频率>堆顶→替换 ➜ 堆中元素即答案

时间复杂度：O(n) 统计 + O(nlogk) 堆操作 = O(nlogk)，优于 O(nlogn)。

4 代码演示
import heapq from collections import Counter def topKFrequent(nums, k): freq = Counter(nums) # 统计频率 heap = [] for num, cnt in freq.items(): heapq.heappush(heap, (cnt, num)) if len(heap) > k: heapq.heappop(heap) # 弹出频率最低的 return [num for _, num in heap]
public int[] topKFrequent(int[] nums, int k) { Map<Integer, Integer> freq = new HashMap<>(); for (int num : nums) freq.merge(num, 1, Integer::sum); PriorityQueue<Integer> pq = new PriorityQueue<>( (a, b) -> freq.get(a) - freq.get(b)); // 按频率小顶堆 for (int num : freq.keySet()) { pq.offer(num); if (pq.size() > k) pq.poll(); } return pq.stream().mapToInt(i -> i).toArray(); }
vector<int> topKFrequent(vector<int>& nums, int k) { unordered_map<int,int> freq; for(int n:nums) freq[n]++; priority_queue<pair<int,int>,vector<pair<int,int>>,greater<>> pq; for(auto& [num,cnt]:freq){ pq.push({cnt,num}); if(pq.size()>(size_t)k) pq.pop(); } vector<int> res; while(!pq.empty()){res.push_back(pq.top().second);pq.pop();} return res; }

6 易错要点
堆的比较依据

堆按频率排序，不是按元素值。Python 中存 (cnt, num)，Java 中比较器用 freq.get(a)。

先统计再入堆

必须先统计完频率，再逐个入堆。不能边遍历原数组边入堆，因为同一元素会出现多次。

小顶堆维护TopK

堆大小超过 k 时弹出堆顶（频率最低的），保留的是频率最高的 k 个。堆顶是这 k 个中频率最低的。

7 扩展思考
  • 桶排序：频率范围有限（1~n），可以用桶排序 O(n)，按频率分桶再从高到低取
  • 快速选择：对频率数组用快速选择找第 k 大频率，O(n) 平均
  • 数据流TopK：如果数据是流式的，堆方法天然支持在线处理
  • 海量数据：MapReduce 分片统计 + 合并，或用 Count-Min Sketch 近似
