#39 组合总和

给定一个无重复元素的数组 candidates 和目标数 target，找出 candidates 中所有可以使数字和为 target 的组合。candidates 中的数字可以无限制重复被选取。

示例 1
输入：candidates = [2,3,6,7], target = 7
输出：[[2,2,3],[7]]
2+2+3=7, 7=7，注意 [2,2,3] 中 2 可重复使用
示例 2
输入：candidates = [2,3,5], target = 8
输出：[[2,2,2,2],[2,3,3],[3,5]]
示例 3
输入：candidates = [2], target = 1
输出：[]
2 题目拆解
无重复数组 ➜ 可重复选取 ➜ 和为 target ➜ 返回所有组合

关键约束：

  • 数字可重复选取 → 递归时传 i 而非 i+1
  • 组合无序：[2,2,3] 和 [2,3,2] 是同一组合 → 用 start 控制顺序
  • 所有数字都是正整数 → 和超过 target 可直接剪枝
3 解题思路

回溯 + 剪枝：从 start 位置开始枚举候选数，加入路径后递归（传 i 而非 i+1 允许重复），若当前和超过 target 则剪枝返回。

从 start 枚举 ➜ 加入路径，sum+=val ➜ sum>target? 剪枝 ➜ sum==target? 收集

剪枝优化：排序后，若 sum + candidates[i] > target，后续更大的数也不用试，直接 break。

4 代码演示
def combinationSum(candidates, target): res = [] candidates.sort() # 排序便于剪枝 def backtrack(start, path, s): if s == target: # 找到组合 res.append(path[:]) return for i in range(start, len(candidates)): if s + candidates[i] > target: # 剪枝 break path.append(candidates[i]) backtrack(i, path, s + candidates[i]) # 传 i，允许重复 path.pop() backtrack(0, [], 0) return res
public List<List<Integer>> combinationSum(int[] candidates, int target) { List<List<Integer>> res = new ArrayList<>(); Arrays.sort(candidates); backtrack(candidates, target, 0, new ArrayList<>(), 0, res); return res; } void backtrack(int[] cands, int target, int start, List<Integer> path, int sum, List<List<Integer>> res) { if (sum == target) { res.add(new ArrayList<>(path)); return; } for (int i = start; i < cands.length; i++) { if (sum + cands[i] > target) break; // 剪枝 path.add(cands[i]); backtrack(cands, target, i, path, sum + cands[i], res); path.remove(path.size() - 1); } }
vector<vector<int>> combinationSum(vector<int>& candidates, int target) { sort(candidates.begin(), candidates.end()); vector<vector<int>> res; vector<int> path; backtrack(candidates, target, 0, path, 0, res); return res; } void backtrack(vector<int>& cands, int target, int start, vector<int>& path, int sum, vector<vector<int>>& res) { if (sum == target) { res.push_back(path); return; } for (int i = start; i < cands.size(); i++) { if (sum + cands[i] > target) break; // 剪枝 path.push_back(cands[i]); backtrack(cands, target, i, path, sum + cands[i], res); path.pop_back(); } }

6 易错要点
传 i 而非 i+1

允许重复选取当前元素，递归调用 backtrack(i, ...)。如果传 i+1 则每个元素只能用一次（变成 #40 组合总和 II）。

排序 + 剪枝

不排序也能做，但排序后可以用 break 提前终止，大幅减少搜索空间。排序是 O(n log n)，剪枝省的时间远超排序。

sum 参数代替每次求和

传 sum 参数比每次 sum(path) 高效得多。注意递归时传 s + candidates[i]，不要修改 s 本身。

7 扩展思考
  • #40 组合总和 II：每个数字只能用一次 + 可能有重复元素，需要排序 + 去重剪枝
  • #216 组合总和 III：只用 1-9，每个用一次，且组合长度固定为 k
  • #377 组合总和 Ⅳ：不同顺序算不同组合，实际是排列问题，用 DP
  • 时间复杂度：最坏 O(2^target)，但剪枝后远小于此
