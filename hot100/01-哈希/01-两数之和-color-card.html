Two Sum两数之和

给定整数数组 nums 和目标值 target，找出和为目标值的两个整数，并返回它们的数组下标。每种输入只对应一个答案，且不能使用同一个元素两次。

输入nums=[2,7,11,15], target=9
输出[0,1]
解释nums[0] + nums[1] = 9

核心模式：查找补数。 Find the missing half from what you have already seen.

题目拆解

01
Input

nums 是整数数组，target 是目标和。

Output

返回两个元素的下标，不是元素值本身。

Constraint

只存在一个有效答案，且同一个元素不能使用两次。

Function

twoSum(nums: List[int], target: int) -> List[int]

解题思路

02
Method 01

暴力枚举

对每个元素，遍历它后面的所有元素，检查两者之和是否等于 target。直观但慢，面试中通常只是铺垫。

Time O(n²)Space O(1)
Method 02 / Recommended

哈希表：用空间换时间

遍历数组时，当前元素 num 需要寻找的是 target - num。如果补数已经在哈希表中，就能 O(1) 得到答案。

Time O(n)Space O(n)

代码与交互演示

03
solution.pyPython
class Solution:
    def twoSum(self, nums, target):
        hash_map = {}

        for i, num in enumerate(nums):
            complement = target - num
            if complement in hash_map:
                return [hash_map[complement], i]
            hash_map[num] = i

        return []

易错点与扩展

04
Pitfall

先查询再插入。 如果先插入，当 nums[i] * 2 == target 时会匹配到自身。

Pitfall

返回顺序。 推荐返回 [hash_map[complement], i]，补数下标在前，当前下标在后。

Pitfall

重复元素。 如 [3,3]，先查后插能自然命中第一个 3，不会冲突。

Extension

如果数组已排序，可以使用双指针，时间 O(n)，空间 O(1)。

Extension

如果要求所有数对，哈希表需要存下标列表，并记录所有匹配。

Pattern

“查找补数”可推广到三数之和、四数相加等题型。
