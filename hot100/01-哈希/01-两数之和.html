#01 两数之和

给定一个整数数组 nums 和目标值 target，找出和为目标值的两个整数，并返回它们的数组下标。你可以假设每种输入只会对应一个答案，且同一元素不能使用两次。

示例 1
输入：nums = [2,7,11,15], target = 9
输出：[0,1]
因为 nums[0] + nums[1] == 9 ，返回 [0, 1]
示例 2
输入：nums = [3,2,4], target = 6
输出：[1,2]
示例 3
输入：nums = [3,3], target = 6
输出：[0,1]
2 题目拆解
输入数组 ➜ 目标值 target ➜ 找两个数之和 ➜ 返回下标

关键约束：

  • 每种输入只有一个答案，不用处理多解
  • 同一个元素不能使用两次 → 先查再存
  • 返回的是下标，不是值
3 解题思路

暴力法 O(n²)：两层循环枚举所有数对 → 太慢，淘汰

哈希法 O(n)：遍历数组时，用哈希表记录已见过的数。对每个 nums[i]，查 target - nums[i] 是否在表中：

遍历 i ➜ 算 complement = target - nums[i] ➜ 查哈希表 ➜ 找到→返回 / 没找到→存入
4 代码演示
def twoSum(nums, target): seen = {} # 哈希表：值→下标 for i, num in enumerate(nums): complement = target - num # 算补数 if complement in seen: # 查表 return [seen[complement], i] # 找到！ seen[num] = i # 先查再存，避免重复使用 return [] # 题目保证有解，不会走到这
public int[] twoSum(int[] nums, int target) { Map<Integer, Integer> seen = new HashMap<>(); for (int i = 0; i < nums.length; i++) { int complement = target - nums[i]; if (seen.containsKey(complement)) { return new int[]{seen.get(complement), i}; } seen.put(nums[i], i); } return new int[]{}; }
vector<int> twoSum(vector<int>& nums, int target) { unordered_map<int, int> seen; for (int i = 0; i < nums.size(); i++) { int complement = target - nums[i]; if (seen.count(complement)) { return {seen[complement], i}; } seen[nums[i]] = i; } return {}; }

6 易错要点
先查再存

必须先查 complement 是否在表中，再把当前数存入。否则 nums=[3], target=6 会错误地使用同一个元素两次。

返回下标

题目要求返回下标不是值，所以哈希表存的是 {值: 下标}，不要搞反。

唯一解

题目保证恰好一个答案，不需要处理无解或多解的情况。找到直接 return 即可。

7 扩展思考
  • 如果数组有序：可以用双指针从两端向中间逼近，空间 O(1)
  • 如果要求返回所有数对：哈希表值改为 List 存多个下标，遍历完再收集
  • 如果数据量极大：哈希表内存可能不够，考虑外部排序 + 双指针
  • 三数之和：排序 + 固定一个数 + 双指针，是本题的自然扩展
