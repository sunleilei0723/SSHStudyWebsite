#42 接雨水

给定 n 个非负整数表示每个宽度为 1 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

示例 1
输入：height = [0,1,0,2,1,0,1,3,2,1,2,1]
输出：6
上面是由数组 [0,1,0,2,1,0,1,3,2,1,2,1] 表示的高度图，在这种情况下，可以接 6 个单位的雨水（蓝色部分表示雨水）
示例 2
输入：height = [4,2,0,3,2,5]
输出：9
2 题目拆解
每根柱子 ➜ 找左右最高 ➜ 算可接水量 ➜ 累加结果

关键约束：

  • 每根柱子宽度为 1，能接的水 = min(左边最高, 右边最高) - 当前高度
  • 如果左右最高中较小值 ≤ 当前高度，该位置接不到水
  • 首尾两根柱子不可能接水（没有更矮的邻居围成凹槽）
3 解题思路

暴力法 O(n²)：每个位置左右扫描找最高 → 太慢

动态规划 O(n)：预处理左右最大值数组 → 空间 O(n)

双指针 O(n)：左右指针向内收缩，维护 maxLeft 和 maxRight，空间 O(1)：

l=0, r=n-1 ➜ 比较 maxL, maxR ➜ 矮侧算水量 ➜ 移动矮侧指针

核心：如果 maxLeft < maxRight，左侧的接水量由 maxLeft 决定（右侧一定有更高的墙挡住）。

4 代码演示
def trap(height): if not height: return 0 l, r = 0, len(height) - 1 maxL, maxR = height[l], height[r] ans = 0 while l < r: if maxL < maxR: # 左侧矮，由 maxL 决定 l += 1 maxL = max(maxL, height[l]) ans += maxL - height[l] # 接水量 else: # 右侧矮，由 maxR 决定 r -= 1 maxR = max(maxR, height[r]) ans += maxR - height[r] return ans
public int trap(int[] height) { if (height.length == 0) return 0; int l = 0, r = height.length - 1; int maxL = height[l], maxR = height[r], ans = 0; while (l < r) { if (maxL < maxR) { l++; maxL = Math.max(maxL, height[l]); ans += maxL - height[l]; } else { r--; maxR = Math.max(maxR, height[r]); ans += maxR - height[r]; } } return ans; }
int trap(vector<int>& height) { if (height.empty()) return 0; int l = 0, r = height.size() - 1; int maxL = height[l], maxR = height[r], ans = 0; while (l < r) { if (maxL < maxR) { l++; maxL = max(maxL, height[l]); ans += maxL - height[l]; } else { r--; maxR = max(maxR, height[r]); ans += maxR - height[r]; } } return ans; }

6 易错要点
先更新max再算水

必须先 maxL = max(maxL, height[l])，再算 ans += maxL - height[l]。顺序反了会少算水。

与盛水容器的区别

#11 盛水容器求的是两根柱子之间的面积，本题求的是所有凹槽能接的水量。思路相似但细节不同。

栈方法

也可以用单调栈：维护递减栈，遇到更高柱子时弹出栈顶计算水量。但双指针更简洁，空间 O(1)。

7 扩展思考
  • 二维接雨水：3D 版本的接雨水，需要用优先队列（最小堆）从边界向内扩展
  • 动态高度：如果柱子高度会变化，需要用线段树或分块维护区间最大值
  • 多轮降雨：如果水可以蒸发再下雨，需要模拟多轮过程
  • 接雨水 II：LeetCode 407，3D 版本，难度更大，需要 BFS + 优先队列
