#25 K个一组翻转链表

给定链表，每 k 个节点一组进行翻转，返回修改后的链表。如果节点数不是 k 的倍数，最后剩余节点保持原样。如 1→2→3→4→5, k=2 → 2→1→4→3→5。

示例 1
输入：head = [1,2,3,4,5], k = 2
输出：[2,1,4,3,5]
2 题目拆解
遍历k个节点 ➜ 判断是否够k个 ➜ 翻转这k个 ➜ 递归处理剩余

关键约束：

  • 不足 k 个的末尾组不翻转，保持原序
  • 不能只交换值，必须修改指针
  • 是 #24 两两交换的通用版本（k=2 时等价）
3 解题思路

递归法：先走 k 步确认够翻转，再反转 k 个，递归处理剩余

检查剩余 ≥ k 个 ➜ 反转 k 个节点 ➜ 原头.next = 递归(剩余) ➜ 返回新头

迭代法：循环处理每组，用 prev 指针连接各组

4 代码演示
def reverseKGroup(head, k): # 1. 检查剩余是否 ≥ k 个 cur = head for _ in range(k): if not cur: return head # 不够k个，不翻转 cur = cur.next # 2. 反转前k个节点 prev, cur = None, head for _ in range(k): nxt = cur.next cur.next = prev prev = cur cur = nxt # 3. 原头连接递归结果 head.next = reverseKGroup(cur, k) return prev # prev是新的头
public ListNode reverseKGroup(ListNode head, int k) { // 1. 检查剩余是否 ≥ k 个 ListNode cur = head; for (int i = 0; i < k; i++) { if (cur == null) return head; cur = cur.next; } // 2. 反转前k个节点 ListNode prev = null; cur = head; for (int i = 0; i < k; i++) { ListNode nxt = cur.next; cur.next = prev; prev = cur; cur = nxt; } // 3. 原头连接递归结果 head.next = reverseKGroup(cur, k); return prev; }
ListNode* reverseKGroup(ListNode* head, int k) { // 1. 检查剩余是否 ≥ k 个 ListNode* cur = head; for (int i = 0; i < k; i++) { if (!cur) return head; cur = cur->next; } // 2. 反转前k个节点 ListNode *prev = nullptr; cur = head; for (int i = 0; i < k; i++) { ListNode* nxt = cur->next; cur->next = prev; prev = cur; cur = nxt; } // 3. 原头连接递归结果 head->next = reverseKGroup(cur, k); return prev; }

6 易错要点
先检查再翻转

必须先走 k 步确认剩余节点 ≥ k，不够则直接返回 head 不翻转。否则会错误翻转末尾不足 k 个的组。

head.next 连接

翻转后原 head 变成组尾，head.next = reverseKGroup(cur, k) 连接下一组。这行必须在翻转之后执行。

返回 prev 不是 head

翻转后 prev 指向新的头节点，head 变成尾节点。返回 prev 才是正确的新头。

7 扩展思考
  • k=2 时：等价于 #24 两两交换链表中的节点
  • 迭代写法：用 dummy + prev 指针循环处理，空间 O(1)，适合面试手写
  • 如果末尾也要翻转：去掉长度检查，直接翻转即可
  • 链表反转是核心操作：掌握 #206 反转链表是做本题的前提
