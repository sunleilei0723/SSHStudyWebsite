#142 环形链表II

给定一个链表，返回链表开始入环的第一个节点。如果链表无环，则返回 null。不允许修改链表。

示例 1
输入：head = [3,2,0,-4], pos = 1
输出：返回索引为 1 的链表节点
链表中有一个环，尾部连接到第二个节点
2题目拆解
有环链表 ➜ 快慢指针找相遇点 ➜ 从head出发新指针 ➜ 再次相遇=入口

关键约束：

  • 在 141 题基础上，不仅要判断有环，还要找环入口
  • 数学推导：相遇点距入口 = head 距入口
  • 不修改链表，O(1) 空间
3解题思路

两阶段法 O(n) O(1)

设 head 到入口距离 a，入口到相遇点距离 b，环长减 b 为 c。快指针走 2(a+b)，慢指针走 a+b，快指针多走 a+b = k×环长。所以 a = c + (k-1)×环长。即从 head 和相遇点同时走，必在入口相遇。

快慢指针找相遇点 ➜ ptr从head出发 ➜ slow从相遇点出发 ➜ ptr==slow → 入口
4代码演示
def detectCycle(head): slow = fast = head # 阶段一：找相遇点 while fast and fast.next: slow = slow.next fast = fast.next.next if slow is fast: break else: return None # 无环 # 阶段二：找入口 ptr = head while ptr is not slow: ptr = ptr.next slow = slow.next return ptr # 环入口
public ListNode detectCycle(ListNode head) { ListNode slow = head, fast = head; while (fast != null && fast.next != null) { slow = slow.next; fast = fast.next.next; if (slow == fast) { ListNode ptr = head; while (ptr != slow) { ptr = ptr.next; slow = slow.next; } return ptr; } } return null; }
ListNode *detectCycle(ListNode *head) { ListNode *slow = head, *fast = head; while (fast && fast->next) { slow = slow->next; fast = fast->next->next; if (slow == fast) { ListNode *ptr = head; while (ptr != slow) { ptr = ptr->next; slow = slow->next; } return ptr; } } return nullptr; }

6易错要点
无环判断

如果快指针走到 null，直接返回 null，不需要进入第二阶段。

ptr 从 head 出发

第二阶段的 ptr 从 head 出发，不是从相遇点出发。两者同速走，相遇处即为入口。

数学证明

a = c + (k-1)×环长，所以 ptr 走 a 步、slow 走 c + (k-1)×环长步后，两者都到入口。

7扩展思考
  • 哈希表法：遍历记录节点引用，第一个重复的即为入口，O(n) 空间
  • 计算环长度：相遇后 slow 再走一圈回到相遇点，步数即环长
  • 为什么 a = c：fast 走了 2(a+b)，slow 走了 a+b，fast 多走 a+b = k×环长 = k×(b+c)，所以 a = c + (k-1)(b+c)
  • 修改链表法：遍历时将 next 指向自身标记，遇到已标记节点即为入口（但违反不修改链表的要求）
