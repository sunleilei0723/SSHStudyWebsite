#24 两两交换链表中的节点

给定一个链表，两两交换其中相邻的节点，并返回交换后的链表头节点。如 1→2→3→4 → 2→1→4→3。不能只是改变节点内部的值，需要实际交换节点。

示例 1
输入：head = [1,2,3,4]
输出：[2,1,4,3]
2 题目拆解
取相邻两个节点 ➜ 交换指针方向 ➜ 连接前驱节点 ➜ 继续下一对

关键约束：

  • 必须交换节点，不能只交换值
  • 节点数为奇数时，最后一个不交换
  • 递归天然适合"处理一对→递归处理剩余"的模式
3 解题思路

递归法：把问题拆成"交换前两个 + 递归处理剩余"

head → 第二个节点 ➜ 第二个节点 → head ➜ head.next = 递归(第三个节点) ➜ 返回新的头

迭代法：用 dummy 节点 + 三指针逐步交换，避免递归栈开销

4 代码演示
def swapPairs(head): # 递归终止：不足两个节点 if not head or not head.next: return head # 交换 head 和 head.next second = head.next head.next = swapPairs(second.next) second.next = head return second # 新的头节点
public ListNode swapPairs(ListNode head) { // 递归终止 if (head == null || head.next == null) { return head; } // 交换 head 和 head.next ListNode second = head.next; head.next = swapPairs(second.next); second.next = head; return second; }
ListNode* swapPairs(ListNode* head) { // 递归终止 if (!head || !head->next) { return head; } // 交换 head 和 head->next ListNode* second = head->next; head->next = swapPairs(second->next); second->next = head; return second; }

6 易错要点
递归顺序

必须先 head.next = swapPairs(second.next) 再 second.next = head，顺序不能反，否则会丢失后续链表。

返回新头

交换后 second 成为新的头节点，不是 head。递归每层都返回交换后的新头。

奇数节点

当节点数为奇数时，最后一个节点不参与交换，递归终止条件 !head || !head.next 自然处理。

7 扩展思考
  • 迭代写法：用 dummy + prev 指针，循环处理每一对，空间 O(1)
  • 扩展为 k 个一组交换：见 #25 K个一组翻转链表，是本题的通用版
  • 只交换值：如果允许只交换值，代码更简单，但面试要求交换节点
  • 递归 vs 迭代：递归简洁但栈空间 O(n)，迭代更安全
