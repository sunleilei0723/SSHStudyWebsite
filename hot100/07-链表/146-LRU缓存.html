#146 LRU缓存

设计一个满足 LRU（最近最少使用）缓存约束的数据结构。实现 get(key) 和 put(key, value)，两个操作都要 O(1) 时间复杂度。容量满时淘汰最久未使用的元素。

示例 1
输入：["LRUCache","put","put","get","put","get","put","get","get","get"], [[2],[1,1],[2,2],[1],[3,3],[2],[4,4],[1],[3],[4]]
输出：[null,null,null,1,null,-1,null,-1,3,4]
2 题目拆解
哈希表 O(1) 查找 ➜ 双向链表维护顺序 ➜ 访问时移到头部 ➜ 满时淘汰尾部

关键约束：

  • get 和 put 必须 O(1) → 哈希表 + 双向链表
  • LRU 淘汰最久未访问的，不是最久未写入的
  • get 也算"使用"，需要将节点移到链表头部
3 解题思路

哈希表 + 双向链表：

哈希表：key → 链表节点 ➜ 双向链表：头部=最近使用 ➜ get：查表+移到头 ➜ put：插入头/更新+淘汰尾

使用虚拟头尾节点（dummy head/tail）简化边界处理，避免空指针判断。

4 代码演示
class LRUCache: def __init__(self, capacity): self.cap = capacity self.cache = {} # key → Node self.head = Node(0,0) # 虚拟头 self.tail = Node(0,0) # 虚拟尾 self.head.next = self.tail self.tail.prev = self.head def get(self, key): if key in self.cache: node = self.cache[key] self._remove(node) self._add_head(node) return node.val return -1 def put(self, key, val): if key in self.cache: self._remove(self.cache[key]) node = Node(key, val) self.cache[key] = node self._add_head(node) if len(self.cache) > self.cap: lru = self.tail.prev self._remove(lru) del self.cache[lru.key]
class LRUCache { Map<Integer, Node> cache = new HashMap<>(); Node head = new Node(0,0), tail = new Node(0,0); int cap; public LRUCache(int capacity) { cap = capacity; head.next = tail; tail.prev = head; } public int get(int key) { if (!cache.containsKey(key)) return -1; Node node = cache.get(key); remove(node); addHead(node); return node.val; } public void put(int key, int val) { if (cache.containsKey(key)) remove(cache.get(key)); Node node = new Node(key, val); cache.put(key, node); addHead(node); if (cache.size() > cap) { Node lru = tail.prev; remove(lru); cache.remove(lru.key); } } }
class LRUCache { int cap; unordered_map<int, Node*> cache; Node *head = new Node(0,0), *tail = new Node(0,0); public: LRUCache(int capacity) : cap(capacity) { head->next = tail; tail->prev = head; } int get(int key) { if (!cache.count(key)) return -1; Node* node = cache[key]; remove(node); addHead(node); return node->val; } void put(int key, int val) { if (cache.count(key)) remove(cache[key]); Node* node = new Node(key, val); cache[key] = node; addHead(node); if (cache.size() > cap) { Node* lru = tail->prev; remove(lru); cache.erase(lru->key); } } };

6 易错要点
get也要移动

get 命中后必须将节点移到链表头部，因为"访问"也算"使用"。忘记移动会导致 LRU 顺序错误。

put已存在的key

put 时如果 key 已存在，要先删除旧节点再插入新节点，不能只更新值。否则链表顺序不对。

虚拟头尾节点

使用 dummy head/tail 避免 null 判断，_remove 和 _add_head 操作始终安全，不会出现空指针。

7 扩展思考
  • LFU缓存：#460 最不经常使用，访问频率最低的先淘汰，比LRU更精细
  • LinkedHashMap：Java 的 LinkedHashMap 天然支持LRU，设置 accessOrder=true 即可
  • 线程安全：生产环境需要加锁或用 ConcurrentHashMap
  • 分布式LRU：Redis 的内存淘汰策略之一，用近似LRU算法
