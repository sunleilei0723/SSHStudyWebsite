#141 环形链表

给你一个链表的头节点 head，判断链表中是否有环。如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。

示例 1
输入：head = [3,2,0,-4], pos = 1
输出：true
链表中有一个环，尾部连接到第二个节点
示例 2
输入：head = [1], pos = -1
输出：false
2题目拆解
单链表 ➜ 判断有无环 ➜ 快慢指针 ➜ 相遇=有环

关键约束：

  • 只需判断是否有环，不需要找环入口
  • 快慢指针：慢指针走1步，快指针走2步
  • 如果有环，快指针一定会追上慢指针
3解题思路

快慢指针法 O(n) O(1)

像操场跑步：快的人每圈都会比慢的人多走一步，最终一定会在环内相遇。如果快指针走到 null，说明无环。

slow 走1步 ➜ fast 走2步 ➜ fast==null? 无环 ➜ slow==fast? 有环
4代码演示
def hasCycle(head): slow = fast = head while fast and fast.next: slow = slow.next # 慢指针走1步 fast = fast.next.next # 快指针走2步 if slow is fast: return True # 相遇 → 有环 return False # fast 到达末尾 → 无环
public boolean hasCycle(ListNode head) { ListNode slow = head, fast = head; while (fast != null && fast.next != null) { slow = slow.next; fast = fast.next.next; if (slow == fast) return true; } return false; }
bool hasCycle(ListNode *head) { ListNode *slow = head, *fast = head; while (fast && fast->next) { slow = slow->next; fast = fast->next->next; if (slow == fast) return true; } return false; }

6易错要点
循环条件

必须检查 fast && fast.next，不能只检查 fast，否则 fast.next.next 可能空指针。

初始位置

slow 和 fast 都从 head 出发，第一次移动后再比较。如果初始就比较，会误判为有环。

一定能追上

快指针每次比慢指针多走1步，距离每轮减1，有限步内必然相遇（不会跳过）。

7扩展思考
  • 哈希表法：遍历并记录每个节点的引用，遇到重复则有环，O(n) 空间
  • 找环入口：142 题，相遇后从 head 再出发一个指针，与 slow 同速走，再次相遇即为入口
  • 计算环长度：相遇后 slow 继续走，再次相遇时走过的步数即为环长
  • 快指针走3步：可能跳过慢指针，不一定能相遇，必须走2步
