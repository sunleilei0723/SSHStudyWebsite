#23 合并K个升序链表

给定一个链表数组，每个链表都已按升序排列，将所有链表合并到一个升序链表中并返回。如 [[1,4,5],[1,3,4],[2,6]] → [1,1,2,3,4,4,5,6]。

2 题目拆解
K个有序链表 ➜ 取各链表头节点 ➜ 选最小值出队 ➜ 重复直到全部合并

关键约束：

  • K 个链表已各自升序排列
  • 每次只需从 K 个头节点中选最小值
  • 核心问题：如何高效维护 K 个候选的最小值
3 解题思路

优先队列（最小堆）O(N log K)：

将K个头节点入堆 ➜ 弹出最小节点 ➜ 将其next入堆 ➜ 重复直到堆空

分治法 O(N log K)：两两合并，类似归并排序的思路

暴力法 O(KN)：逐个合并，每次合并两个 → 太慢

4 代码演示
import heapq def mergeKLists(lists): # 最小堆：(val, idx, node) heap = [] for i, node in enumerate(lists): if node: heapq.heappush(heap, (node.val, i, node)) dummy = cur = ListNode(0) while heap: val, i, node = heapq.heappop(heap) cur.next = node cur = cur.next if node.next: heapq.heappush(heap, (node.next.val, i, node.next)) return dummy.next
public ListNode mergeKLists(ListNode[] lists) { // 最小堆 PriorityQueue<ListNode> pq = new PriorityQueue<>( (a, b) -> a.val - b.val); for (ListNode node : lists) { if (node != null) pq.offer(node); } ListNode dummy = new ListNode(0), cur = dummy; while (!pq.isEmpty()) { ListNode node = pq.poll(); cur.next = node; cur = cur.next; if (node.next != null) pq.offer(node.next); } return dummy.next; }
ListNode* mergeKLists(vector<ListNode*>& lists) { // 最小堆 auto cmp = [](ListNode* a, ListNode* b) { return a->val > b->val; }; priority_queue<ListNode*, vector<ListNode*>, decltype(cmp)> pq(cmp); for (auto node : lists) { if (node) pq.push(node); } ListNode *dummy = new ListNode(0), *cur = dummy; while (!pq.empty()) { ListNode* node = pq.top(); pq.pop(); cur->next = node; cur = cur->next; if (node->next) pq.push(node->next); } return dummy->next; }

6 易错要点
Python 堆的 tie-breaker

Python 的 heapq 比较元组时，val 相同会比较第二个元素。必须加 idx 作为 tie-breaker，否则 ListNode 不可比较会报错。

空链表跳过

初始化堆时，lists 中可能有 null 链表，必须判断 if node 再入堆，否则空节点入堆会出错。

堆的大小

堆中最多 K 个元素，每次操作 O(log K)。总共有 N 个节点，所以总时间 O(N log K)，远优于暴力 O(KN)。

7 扩展思考
  • 分治法：两两配对合并，递归缩小规模，空间 O(log K)（递归栈），不需要堆
  • 暴力两两合并：O(KN)，每次合并结果与下一个链表，最差情况
  • K=2 时：退化为 #21 合并两个有序链表
  • 外部排序：K 路归并是外部排序的核心操作，本题是其链表版本
