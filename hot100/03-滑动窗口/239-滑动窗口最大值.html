#239 滑动窗口最大值

给定整数数组 nums 和窗口大小 k，滑动窗口从左到右移动，返回每个窗口中的最大值组成的数组。

2 题目拆解
输入数组 + k ➜ 窗口滑动 ➜ 每步求最大值 ➜ 返回结果数组

关键约束：

  • 窗口大小固定为 k，每次右移一位
  • 共产生 n - k + 1 个窗口
  • 需要高效获取每个窗口最大值 → 暴力 O(nk) 太慢
  • 1 ≤ k ≤ nums.length，k 可能为 1 或 n
3 解题思路

暴力法 O(nk)：每个窗口遍历找最大 → 太慢，淘汰

单调队列 O(n)：维护一个单调递减的双端队列，队首始终是当前窗口最大值。新元素入队时，从队尾弹出所有比它小的元素（它们永远不可能成为最大值）：

新元素入队 ➜ 弹出队尾更小元素 ➜ 队首过期？弹出 ➜ 队首即最大值
4 代码演示
from collections import deque def maxSlidingWindow(nums: List[int], k: int) -> List[int]: dq = deque() # 存下标，单调递减 res = [] for i, num in enumerate(nums): while dq and nums[dq[-1]] < num: dq.pop() # 弹出队尾更小元素 dq.append(i) if dq[0] <= i - k: dq.popleft() # 队首过期，弹出 if i >= k - 1: res.append(nums[dq[0]]) # 队首即最大值 return res
public int[] maxSlidingWindow(int[] nums, int k) { Deque<Integer> dq = new ArrayDeque<>(); int[] res = new int[nums.length - k + 1]; for (int i = 0; i < nums.length; i++) { while (!dq.isEmpty() && nums[dq.peekLast()] < nums[i]) { dq.pollLast(); } dq.offerLast(i); if (dq.peekFirst() <= i - k) dq.pollFirst(); if (i >= k - 1) res[i - k + 1] = nums[dq.peekFirst()]; } return res; }
vector<int> maxSlidingWindow(vector<int>& nums, int k) { deque<int> dq; // 存下标 vector<int> res; for (int i = 0; i < nums.size(); i++) { while (!dq.empty() && nums[dq.back()] < nums[i]) { dq.pop_back(); } dq.push_back(i); if (dq.front() <= i - k) dq.pop_front(); if (i >= k - 1) res.push_back(nums[dq.front()]); } return res; }

6 易错要点
队列存下标而非值

单调队列中存储的是下标，不是值。存下标才能判断队首是否过期（dq[0] <= i - k），存值无法判断。

弹出条件是 < 而非 <=

队尾弹出条件是 nums[dq[-1]] < num，不是 <=。相等时不弹出，保证相同值的元素按顺序留在队列中，不会遗漏。

先弹出再入队

必须先从队尾弹出更小元素，再把当前下标入队。顺序反了会导致刚入队的元素被自己弹出。

7 扩展思考
  • 滑动窗口最小值：将单调递减队列改为单调递增队列，队首即最小值
  • 滑动窗口中位数：用两个堆（大顶堆+小顶堆）或有序集合维护，比单调队列复杂
  • 双端队列的其他应用：求柱状图最大矩形面积（#84），也是单调栈/队列的经典应用
  • 与优先队列的对比：堆也能 O(n log n) 解决，但删除过期元素需要懒删除；单调队列 O(n) 更优
