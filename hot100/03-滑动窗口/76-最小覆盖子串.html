#76 最小覆盖子串

给定字符串 s 和 t，返回 s 中涵盖 t 所有字符（含频率）的最小子串。如果不存在返回空串。

示例 1
输入：s = "ADOBECODEBANC", t = "ABC"
输出："BANC"
最小覆盖子串 "BANC" 包含 t 中所有字符
示例 2
输入：s = "a", t = "a"
输出："a"
2 题目拆解
输入 s 和 t ➜ 统计 t 的字符频率 ➜ 滑动窗口覆盖 t ➜ 收缩找最小

关键约束：

  • 覆盖 = 窗口内每个字符数量 ≥ t 中对应数量
  • 找最小覆盖子串 → 满足条件后要收缩窗口
  • t 中可能有重复字符，频率必须全部满足
  • 区分大小写，'A' 和 'a' 是不同字符
3 解题思路

核心：变长滑动窗口 + need 字典 + valid 计数

用 need 字典记录 t 中每个字符的需求数，window 记录窗口内字符数，valid 记录已满足需求的字符种类数。当 valid == need.size() 时窗口满足条件，尝试收缩：

right 扩展窗口 ➜ 更新 window + valid ➜ valid 满足？ ➜ left 收缩找最小
4 代码演示
def minWindow(s: str, t: str) -> str: need = Counter(t) window = {} valid = 0 # 满足需求的字符种类数 left = 0 start, min_len = 0, float('inf') for right, ch in enumerate(s): if ch in need: window[ch] = window.get(ch, 0) + 1 if window[ch] == need[ch]: valid += 1 while valid == len(need): # 窗口满足条件 if right - left + 1 < min_len: start, min_len = left, right - left + 1 d = s[left] if d in need: if window[d] == need[d]: valid -= 1 window[d] -= 1 left += 1 return "" if min_len == float('inf') else s[start:start+min_len]
public String minWindow(String s, String t) { Map<Character, Integer> need = new HashMap<>(); Map<Character, Integer> window = new HashMap<>(); for (char c : t.toCharArray()) need.merge(c, 1, Integer::sum); int valid = 0, left = 0, start = 0, minLen = Integer.MAX_VALUE; for (int right = 0; right < s.length(); right++) { char ch = s.charAt(right); if (need.containsKey(ch)) { window.merge(ch, 1, Integer::sum); if (window.get(ch).equals(need.get(ch))) valid++; } while (valid == need.size()) { if (right - left + 1 < minLen) { start = left; minLen = right - left + 1; } char d = s.charAt(left++); if (need.containsKey(d)) { if (window.get(d).equals(need.get(d))) valid--; window.merge(d, -1, Integer::sum); } } } return minLen == Integer.MAX_VALUE ? "" : s.substring(start, start + minLen); }
string minWindow(string s, string t) { unordered_map<char, int> need, window; for (char c : t) need[c]++; int valid = 0, left = 0, start = 0, minLen = INT_MAX; for (int right = 0; right < s.size(); right++) { if (need.count(s[right])) { window[s[right]]++; if (window[s[right]] == need[s[right]]) valid++; } while (valid == need.size()) { if (right - left + 1 < minLen) { start = left; minLen = right - left + 1; } if (need.count(s[left])) { if (window[s[left]] == need[s[left]]) valid--; window[s[left]]--; } left++; } } return minLen == INT_MAX ? "" : s.substr(start, minLen); }

6 易错要点
valid 判断条件

valid == need.size() 才表示所有字符种类都满足。不是 window == need，因为窗口中某字符可能比 need 多，但仍然满足条件。

收缩时 valid 递减时机

只有当 window[d] == need[d] 时才 valid--，即恰好从"满足"变为"不满足"。如果 window[d] > need[d]，减少后仍然满足，valid 不变。

只处理 need 中的字符

不在 need 中的字符（如 'D','O'）入窗/出窗时不影响 window 和 valid，直接跳过即可。不要把无关字符加入 window。

7 扩展思考
  • 如果 t 中有重复字符：need 记录频率而非集合，valid 判断 window[ch] == need[ch] 而非仅存在
  • 优化：过滤 s：先过滤掉 t 中不存在的字符，只保留有效字符和索引，减少遍历次数
  • 多模式匹配：如果需要同时匹配多个 t，可考虑 AC 自动机
  • 与 #438 的对比：#438 是定长窗口（窗口大小= p 长度），本题是变长窗口（需要收缩找最小）
