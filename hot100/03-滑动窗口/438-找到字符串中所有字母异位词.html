#438 找到字符串中所有字母异位词

给定字符串 s 和 p，找到 s 中所有 p 的字母异位词的起始索引。字母异位词指字母相同但排列不同的字符串。

示例 1
输入：s = "cbaebabacd", p = "abc"
输出：[0,6]
起始索引等于 0 的子串是 "cba"，起始索引等于 6 的子串是 "bac"
示例 2
输入：s = "abab", p = "ab"
输出：[0,1,2]
2 题目拆解
输入 s 和 p ➜ 统计 p 的字符频率 ➜ 滑动窗口匹配 ➜ 返回所有起始索引

关键约束：

  • 字母异位词 = 字母组成相同，顺序不同
  • 窗口大小固定为 len(p)，是定长滑动窗口
  • s 和 p 仅包含小写英文字母 → 可用数组代替哈希表
  • 返回所有匹配的起始索引
3 解题思路

暴力法 O(n·m)：每个位置截取子串排序比较 → 太慢，淘汰

滑动窗口 + 字符计数 O(n)：用计数数组记录 p 的字符频率，维护窗口内的字符频率。窗口滑动时 O(1) 更新计数，用 match 变量跟踪匹配的字符种类数：

统计 p 的字符频率 ➜ 右指针扩展窗口 ➜ 窗口满时比较计数 ➜ 左指针收缩+更新
4 代码演示
def findAnagrams(s: str, p: str) -> List[int]: need = Counter(p) # p 的字符频率 window = Counter() # 窗口字符频率 res = [] for right, ch in enumerate(s): window[ch] += 1 # 右字符入窗 if right >= len(p): # 窗口超长 left_ch = s[right - len(p)] window[left_ch] -= 1 # 左字符出窗 if window[left_ch] == 0: del window[left_ch] if window == need: # 频率一致→找到！ res.append(right - len(p) + 1) return res
public List<Integer> findAnagrams(String s, String p) { int[] need = new int[26], window = new int[26]; for (char c : p.toCharArray()) need[c - 'a']++; List<Integer> res = new ArrayList<>(); for (int i = 0; i < s.length(); i++) { window[s.charAt(i) - 'a']++; if (i >= p.length()) { window[s.charAt(i - p.length()) - 'a']--; } if (Arrays.equals(need, window)) { res.add(i - p.length() + 1); } } return res; }
vector<int> findAnagrams(string s, string p) { int need[26] = {}, window[26] = {}; for (char c : p) need[c - 'a']++; vector<int> res; for (int i = 0; i < s.size(); i++) { window[s[i] - 'a']++; if (i >= p.size()) { window[s[i - p.size()] - 'a']--; } if (memcmp(need, window, sizeof(need)) == 0) { res.push_back(i - p.size() + 1); } } return res; }

6 易错要点
窗口大小固定

本题窗口大小 = len(p)，是定长窗口。当 right >= len(p) 时必须移除左端字符，否则窗口会越变越大。

Counter 比较技巧

Python 中 window == need 可以直接比较两个 Counter，但要注意 del 计数为 0 的键，否则 {a:0} 和 {} 不相等。

起始索引计算

找到匹配时，起始索引 = right - len(p) + 1，不是 right - len(p)。差一个位置会导致结果全部偏移。

7 扩展思考
  • 优化：用 match 变量：维护匹配字符种类数，避免每次比较整个计数数组，O(1) 判断是否匹配
  • 如果字符集很大：用 HashMap 代替数组，但逻辑不变
  • 变位词判断：排序后比较 O(m log m)，不如计数法 O(m) 高效
  • 与 #76 最小覆盖子串的关系：本题是定长窗口版本，#76 是变长窗口版本
