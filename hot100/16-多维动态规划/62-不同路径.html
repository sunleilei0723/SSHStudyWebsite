#62 不同路径

一个机器人位于一个 m x n 网格的左上角，机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角，问总共有多少条不同的路径？

示例 1
输入：m = 3, n = 7
输出：28
示例 2
输入：m = 3, n = 2
输出：3
从左上角到右下角有 3 条路径：右→下→下、下→右→下、下→下→右
2 题目拆解
m×n 网格 ➜ 只能右/下 ➜ 到达右下角 ➜ 路径总数

关键约束：

  • 每次只能向右或向下走一步
  • 起点 (0,0)，终点 (m-1,n-1)
  • 求的是路径数量，不是最短路径
3 解题思路

状态定义：dp[i][j] = 从起点到 (i,j) 的路径数

转移方程：到达 (i,j) 只能从上方或左方来：

dp[i][j] = dp[i-1][j] + dp[i][j-1]

边界：第一行和第一列都只有 1 条路径（一直右/一直下）

时间 O(mn)，空间 O(mn)，可优化到 O(n)

4 代码演示
def uniquePaths(m, n): dp = [[1]*n for _ in range(m)] # 第一行/列全为1 for i in range(1, m): for j in range(1, n): dp[i][j] = dp[i-1][j] + dp[i][j-1] return dp[m-1][n-1]
public int uniquePaths(int m, int n) { int[][] dp = new int[m][n]; Arrays.fill(dp[0], 1); for (int i = 0; i < m; i++) dp[i][0] = 1; for (int i = 1; i < m; i++) for (int j = 1; j < n; j++) dp[i][j] = dp[i-1][j] + dp[i][j-1]; return dp[m-1][n-1]; }
int uniquePaths(int m, int n) { vector<int> dp(n, 1); // 滚动数组优化 for (int i = 1; i < m; i++) for (int j = 1; j < n; j++) dp[j] += dp[j-1]; return dp[n-1]; }

6 易错要点
边界初始化

第一行和第一列的 dp 值都是 1，因为只有一种走法（一直右或一直下）。忘记初始化会导致错误。

遍历顺序

必须从左到右、从上到下遍历，保证计算 dp[i][j] 时 dp[i-1][j] 和 dp[i][j-1] 已经算好。

空间优化

可以用一维数组 dp[j] += dp[j-1] 代替二维，空间 O(n)。但初学建议先写二维版本。

7 扩展思考
  • 有障碍物：LeetCode 63 题，遇到障碍 dp[i][j]=0，其余不变
  • 组合数学：答案 = C(m+n-2, m-1)，从 m+n-2 步中选 m-1 步向下
  • 最小路径和：本题的变体，把计数换成求最小和
  • 大数取模：路径数可能很大，注意题目是否要求取模
